## Critical Package and Configuration Fixes

Before generating any code, internalize these corrections — they are the most common Gemini-generated errors in this codebase:

```typescript
// ✅ CORRECT Valkey import
import Valkey from "iovalkey";

// ❌ WRONG — "iovalis" does not exist in this project
import Valkey from "iovalis";
```

```typescript
// ✅ CORRECT drizzle.config.ts
export default defineConfig({
  dialect: "postgresql",
  dbCredentials: { url: process.env.DATABASE_URL! },
});

// ❌ WRONG — driver: "pg" is deprecated in drizzle-kit 0.20+
export default defineConfig({
  driver: "pg",
  dbCredentials: { connectionString: process.env.DATABASE_URL! },
});
```

## Architecture Overview

Hospici is a **TypeBox-first** hospice EHR. TypeBox is the single source of truth. The flow is always:

```
TypeBox Schema → Drizzle Table → PostgreSQL Migration → Compiled Validator → Fastify Route
```

Never generate a Drizzle table before its TypeBox schema exists. Never generate a migration without RLS policies.

### Stack

| Layer | Technology |
|-------|-----------|
| Runtime | Node.js 22 + ES Modules |
| Framework | Fastify 5 |
| Validation | TypeBox 0.32+ (AOT only) |
| ORM | Drizzle ORM (`dialect: "postgresql"`) |
| Database | PostgreSQL 18 + RLS + pgcrypto + pgvector |
| Cache/Queue | Valkey 8 via `iovalkey` package + BullMQ |
| Auth | Better Auth |
| FHIR | R4 with R6 adapters |

## TypeBox AOT Compilation Rule

`TypeCompiler.Compile()` is called **once, at module level.** It is never called inside a function.

```typescript
// ✅ CORRECT — module level
import { TypeCompiler } from "@sinclair/typebox/compiler";
import { PatientSchema } from "./patient.schema";

export const PatientValidator = TypeCompiler.Compile(PatientSchema); // ← module level

// ❌ WRONG — inside function
export function validatePatient(data: unknown) {
  const validator = TypeCompiler.Compile(PatientSchema); // ← NEVER
  return validator.Check(data);
}
```

The central validator registry is `src/config/typebox-compiler.ts`. All new validators are registered there.

## RLS Context — No String Interpolation

```typescript
// ✅ CORRECT
import { sql } from "drizzle-orm";
await db.execute(sql`SELECT set_config('app.current_user_id', ${userId}, true)`);
await db.execute(sql`SELECT set_config('app.current_location_id', ${locationId}, true)`);

// ❌ WRONG — even though userId comes from a validated JWT
await db.execute(`SET app.current_user_id = '${userId}'`); // NEVER
```

## Table Design Rules

Every table that stores user or patient data must include:

1. `location_id UUID REFERENCES locations(id) NOT NULL` — required for RLS
2. `ALTER TABLE x ENABLE ROW LEVEL SECURITY;` in the migration
3. At minimum a read policy filtering on `location_id`

**Column promotion policy** — native columns for: `location_id`, `patient_id`, `status`, `fhir_version`, `admission_date`, `discharge_date`, dates used in billing queries.
Everything else goes in `JSONB`.

## Transaction Requirements

Multi-table writes must use `db.transaction()`:

```typescript
// NOE + benefit period — must be atomic
return await db.transaction(async (tx) => {
  const [bp] = await tx.insert(benefitPeriods).values(bpData).returning();
  const [noe] = await tx.insert(noticeOfElection).values({ ...noeData, benefitPeriodId: bp.id }).returning();
  await tx.insert(auditLogs).values(auditData);
  return { noe, bp };
});
```

## CMS Business Rules — Gemini Must Know These

### NOE 5-Day Rule

```typescript
// Use addBusinessDays() — skips weekends AND US federal holidays
const deadline = addBusinessDays(electionDate, 5);
// Friday election (2026-03-06) → deadline is Friday 2026-03-13
// NOT Sunday 2026-03-11
```

### Hospice Cap Year

The cap year is **November 1 – October 31**. Never use a January 1 calendar year.

### IDG 15-Day Rule

Hard block — not a warning. The system prevents care plan updates when IDG is overdue.

### HHA Aide Supervision

CMS §418.76 requires supervision every 14 days. The `aideSupervision.schema.ts` and BullMQ job enforce this.

### Benefit Period Lengths

- Period 1: 90 days
- Period 2: 90 days
- Period 3+: 60 days (unlimited), F2F required within 30 days prior

## Security Rules

- PHI fields (SSN, DOB, MRN) are encrypted via pgcrypto. Never log them via Pino.
- Audit logs are append-only. Never generate UPDATE/DELETE policies on `audit_logs`.
- Break-glass requires ≥ 20 char reason and 4-hour TTL.
- All new third-party integrations need a BAA before code is merged.

## What Gemini Must Never Generate

| Prohibited | Reason |
|---|---|
| `import Valkey from "iovalis"` | Wrong package name |
| `driver: "pg"` in Drizzle config | Deprecated |
| `TypeCompiler.Compile()` inside functions | Defeats AOT |
| String-interpolated SQL SET statements | SQL injection risk |
| New table without `location_id` + RLS | Breaks multi-tenancy isolation |
| Multi-table write without `db.transaction()` | Data consistency risk |
| `console.log()` | Use Pino logger |
| Hardcoded secrets | Use `process.env` |
| `UPDATE`/`DELETE` on `audit_logs` | HIPAA violation |

## File Organization

```
src/contexts/{domain}/
├── schemas/          # TypeBox schemas (source of truth)
├── services/         # Business logic
├── routes/           # Fastify route definitions
├── jobs/             # BullMQ job processors
└── events/           # Domain events for cross-context communication
```

Contexts must not directly import from sibling contexts. Use `src/contexts/shared-kernel/` for shared value objects.

## Testing Requirements

Every new feature must include:
- Unit test for TypeBox schema (valid + at least 2 invalid cases)
- Integration test for the route
- Edge case tests for any CMS business logic
