# CI/CD Integration

GitHub Actions workflows for Hospici code review automation.
Two workflows: PR review (blocking) and daily cron (reporting).

---

## Workflow 1: PR Review (`.github/workflows/hospici-review-pr.yml`)

```yaml
name: Hospici Code Review — PR

on:
  pull_request:
    types: [opened, synchronize, reopened]
    branches: [main, develop]

permissions:
  contents: read
  issues: write
  pull-requests: write
  checks: write

jobs:
  review:
    name: Hospici Review
    runs-on: ubuntu-latest
    timeout-minutes: 15

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for git diff

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run Biome check
        id: biome
        run: |
          npx biome ci . --reporter=json > biome-output.json 2>&1 || true
          echo "biome_errors=$(cat biome-output.json | jq '.summary.errorCount // 0')" >> $GITHUB_OUTPUT

      - name: Install Claude CLI
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Hospici Review
        id: review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          PR_NUMBER: ${{ github.event.pull_request.number }}
          IS_CI: "true"
          REVIEW_SCOPE: "pr-diff"
        run: |
          # Get changed files
          git diff --name-only origin/${{ github.base_ref }}...HEAD > changed-files.txt
          
          # Run review on changed files
          claude -p "/review $(cat changed-files.txt | tr '\n' ' ') --pre-commit" \
            --output-format json > review-output.json 2>&1 || true
          
          # Extract critical count
          CRITICAL=$(cat review-output.json | jq '.critical_count // 0')
          echo "critical_count=$CRITICAL" >> $GITHUB_OUTPUT
          
          # Extract full report
          cat review-output.json | jq -r '.report // ""' > review-report.md

      - name: Post PR Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs')
            const report = fs.existsSync('review-report.md') 
              ? fs.readFileSync('review-report.md', 'utf8') 
              : 'Review completed.'
            const criticalCount = parseInt('${{ steps.review.outputs.critical_count }}') || 0
            const biomeErrors = parseInt('${{ steps.biome.outputs.biome_errors }}') || 0
            
            const status = criticalCount > 0 || biomeErrors > 0 ? '🔴 Issues Found' : '✅ Clean'
            
            const body = `## Hospici Code Review — ${status}
            
            <details>
            <summary>View full report (${criticalCount} critical, ${biomeErrors} biome errors)</summary>
            
            \`\`\`
            ${report}
            \`\`\`
            </details>
            
            *Generated by hospici-review v2.0*`
            
            // Find existing comment and update, or create new
            const { data: comments } = await github.rest.issues.listComments({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
            })
            
            const existing = comments.find(c => c.body.includes('hospici-review v2.0'))
            
            if (existing) {
              await github.rest.issues.updateComment({
                owner: context.repo.owner,
                repo: context.repo.repo,
                comment_id: existing.id,
                body,
              })
            } else {
              await github.rest.issues.createComment({
                owner: context.repo.owner,
                repo: context.repo.repo,
                issue_number: context.issue.number,
                body,
              })
            }

      - name: Label PR
        uses: actions/github-script@v7
        with:
          script: |
            const criticalCount = parseInt('${{ steps.review.outputs.critical_count }}') || 0
            const biomeErrors = parseInt('${{ steps.biome.outputs.biome_errors }}') || 0
            
            const labels = []
            if (criticalCount > 0) labels.push('review:critical')
            else if (biomeErrors > 0) labels.push('review:warning')
            else labels.push('review:clean')
            
            await github.rest.issues.addLabels({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              labels,
            })

      - name: Fail on Critical
        if: steps.review.outputs.critical_count > 0 || steps.biome.outputs.biome_errors > 0
        run: |
          echo "❌ Hospici review found ${{ steps.review.outputs.critical_count }} critical issues"
          echo "❌ Biome found ${{ steps.biome.outputs.biome_errors }} errors"
          exit 1

      - name: Upload review artifacts
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: hospici-review-${{ github.run_number }}
          path: |
            review-output.json
            review-report.md
            biome-output.json
          retention-days: 30
```

## Workflow 2: Daily Cron (`.github/workflows/hospici-review-daily.yml`)

```yaml
name: Hospici Code Review — Daily

on:
  schedule:
    - cron: '0 6 * * *'   # 06:00 UTC = ~10pm PST / 11pm PDT
  workflow_dispatch:       # Allow manual trigger

permissions:
  contents: write          # To commit REVIEW_LOG.md updates
  issues: write

jobs:
  daily-review:
    name: Daily Full Scan
    runs-on: ubuntu-latest
    timeout-minutes: 30

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Install Claude CLI
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Full Hospici Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          IS_CI: "true"
          REVIEW_SCOPE: "full-repo"
          REVIEW_TRIGGER: "daily-cron"
        run: |
          claude -p "/review" --output-format json > daily-review-output.json 2>&1 || true

      - name: Update REVIEW_LOG.md
        run: |
          cat daily-review-output.json | jq -r '.review_log_entry // ""' >> REVIEW_LOG.md

      - name: Open GitHub Issues for new findings
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs')
            const output = JSON.parse(fs.readFileSync('daily-review-output.json', 'utf8'))
            const findings = output.new_findings || []
            
            for (const finding of findings) {
              if (finding.severity === 'critical' || finding.severity === 'warning') {
                await github.rest.issues.create({
                  owner: context.repo.owner,
                  repo: context.repo.repo,
                  title: `[HOSPICI-REVIEW] ${finding.severity.toUpperCase()} — ${finding.title}`,
                  body: finding.issue_body,
                  labels: [`review:${finding.severity}`, `category:${finding.category}`],
                })
              }
            }

      - name: Commit REVIEW_LOG.md
        run: |
          git config user.name "hospici-review[bot]"
          git config user.email "hospici-review[bot]@users.noreply.github.com"
          git add REVIEW_LOG.md
          git add .claude/skills/hospici-review/references/learned-rules.yaml
          git diff --cached --quiet || git commit -m "chore: daily review log + learned rules update [skip ci]"
          git push
```

## Required GitHub Secrets

Add these in your repo Settings → Secrets and Variables → Actions:

| Secret | Value |
|---|---|
| `ANTHROPIC_API_KEY` | Your Anthropic API key |

The `GITHUB_TOKEN` is automatically provided by GitHub Actions.

---

## Required GitHub Labels

Create these labels in your repo (Settings → Labels):

| Label | Color | Description |
|---|---|---|
| `review:critical` | `#d73a4a` | Critical review findings |
| `review:warning` | `#e4e669` | Warning review findings |
| `review:clean` | `#0075ca` | Review passed clean |
| `category:phi` | `#b60205` | PHI-related finding |
| `category:auth` | `#d93f0b` | Authentication finding |
| `category:drizzle` | `#0052cc` | Database finding |
| `category:regulatory` | `#5319e7` | Regulatory compliance finding |

---

## Branch Protection Rules

Recommended branch protection for `main`:
- Require status check: `Hospici Review` to pass
- Require PR before merging
- Dismiss stale reviews when new commits pushed
